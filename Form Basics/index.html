Form Basics

Intermediate HTML and CSS Course

  • The Form Element
    • Container Element
    • Wraps all of the inputs a user will interact with on a form
    • Accepts two essential attributes:
      1. action - takes a URL value that tells the form where it should send its data to be processed. (Hook backend systems up to a frontend forms using this attribute)
      2. method - tells the browser which HTTP request methods it should use to submit the form
        • GET - used when you want to retrieve something from a server.
          Example: Google uses a GET request when you search
        • POST - used when you want to change something on the server.
          Example: when a user makes an account or makes a payment on a website.
  • Form Controls
    • text boxes
    • dropdowns
    • checkboxes
    • buttons
  • The input element - Accepts a type attribute which tells the browser what type of data it should expect & how it should render the input element
    • Text input - ('< input type="text" >') - accepts any text input
    • Labels - ('< label for="">< /label>') input on its own isn't very useful if the user won't know what kind of data they are supposed to provide.
      Input associated with a label needs an 'id' attribute with the same value as the label's 'for' attribute
    • Placeholder attribute - can guide users on what to enter in form elements
    • Name attribute - we use labels so users understand what data needs to be entered into an input field. We also need to let the backend, where we send our data, know what each piece of data represents.
      The name attribute serves as a reference to the data inputted into a form. A variable name for the input. Form input should always have a name attribute otherwise, it will be ignored when the form is submitted.
    • The specialized input type attributes
      1. Email inputs - specialized text inputs just for email addresses
      2. Password inputs - specialized text input making the inputted data with another character, usually an asterisk or bullet point
      3. Number inputs - only accept number values and ignores any other characters the user tries to enter
      4. Date input - provides a better user experience for choosing dates by rendering a date picker calendar
      5. Textarea - an input box that can accept text that spans multiple lines. Can also be resized by user by clicking and dragging bottom right corner
          6. Textarea accept 2 unique attributes:
        • rows (height)
        • cols (width)
  • Selection elements - for users to select a value from a predefined list
    • Select dropdown
      1. '< select >
      2. '< option > - all opetion elements should have a value attribute
      3. < optgroup > - can split the list of options into groups with
      4. < label > - optgroup element takes a label attribute
      5. Radio buttons - use when you have a smaller list of 5 or fewer.
        input type='radio'
      6. < checked > - default selection of radio button & checkboxes
      7. checkboxes - like radio buttons but alow multiple options to be selected at once. Can also use if we want users to toggle if they want something to be true or false. Like signing up to a newsletter.
        input type='checkbox'
  • Buttons - clickable elements users can interact with to submit forms or trigger other actions.
    • use < button >
    • Button elements can also accept a type attribute. There are 3 types of buttons
      • Submit buttons
      • Reset button
      • Generic button

    Organizing form elements

  • Fieldset element - container element that allows us to group related form inputs into one logical unit. Use < fieldset >. Whatever form inputs you want grouped together go within the opening and closing fieldset tags
  • Legend - used to give field sets a heading or caption so the user can see what a grouping of inputs is for. Use < legend > elements with the test you want to display within it's opening and closing tags. A legend should always come right after an opening fieldset tag.
