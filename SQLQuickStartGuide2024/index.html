SQL - Structured Query Language

    Key SQL Statements

  • SELECT - used to retrieve data from a database (Can be as simple as selecting specific column from a table or as complex as combining data from multiple tables based on logical relationships)

      SELECT basic syntax: begins with 'SELECT', followed by the columns to retrieve, the 'FROM' clause indicating the table to query, and an ~*optional*~ 'WHERE' to clause to filter records

    • SELECT column1, column2 FROM table_name
      WHERE condition;

  • INSERT INTO - to add new records to a table (specifies the table to insert the data into and the values to add)
  • UPDATE - to modify existing records (identifies the table and the specific records to update, based on a condition, and specifies the new values)
  • DELETE - to remove records from a table (removes records that match a specified condition)

    Clauses and Conditions:

  • WHERE - specifies a condition for filtering records and can use logical operators to combine conditions

      • Logical Operators

    • AND
    • OR
    • NOT

  • ORDER BY - determines the order in which the results are returned, based on one or more columns

    • ASC - ascending
    • DESC - descending

  • GROUP BY - aggregates rows that have the same values in specified columns into summary rows

      • Examples:

    • sum
    • average

  • HAVING - similar to WHERE clause but used exclusively to filter rows after they have been grouped.

    SQL Data Types

  • numeric
  • data and time
  • string (character and/or textual)
  • +more

    Best Practices

  • always use explicit column names in your SELECT statements
  • use the WHERE clause to filter rows efficiently
  • make use of comments to enhance readability and maintainability of your SQL scripts

    The Difference Between SQL and NoSQL

  • SQL
  • SQL databases - a.k.a relational databases - are structured in a way that allows for the storage and retrieval of data in relation to other data in said database

      • Examples of popular SQL databases

    • MySQL
    • PostgreSQL
    • Oracle
    • Microsoft SQL Server

    • These systems are widely used in scenarios where data integrity and relationships are crucial (ie: financial records, customer relationship management (CRM) systems, and other applications where transactions are complex and interconnected)

  • These databases are table-bases, meaning data is organized into rows and columns, where each row represents a record and each column represents a data attribute
  • SQL - the language used to communicate with these databases is highly structured allowing for complex queries and transactions
  • This structure ensures data integrity and relationships through predefined schemas and rules (ie: foreign keys) which enforce the relational aspect of the database.
  • SQL databases are traditionally scaled by enhancing the horse-power of the hardware, known as vertical scaling, which can be expensive and has physical limitations.

  • NoSQL
  • NoSQL databases are designed to handle a wide variety of data types

      • NoSQL data types:

    • structured
    • semi-structured
    • unstructured

  • NoSQL databases do not rely on a fixed table structure and are more flexable which allows for easy storage and retrieval. This makes NoSQL databases ideal for handling big data and real-time web applications.

      • Types of NoSQL databases


      each is optimized for specific kinds of data and use cases, from storing large volumes of simple key-value pairs to efficiently managing highly connected data for social networks or recommendation systems

      Document Databases

    • MongoDB
    • Couchbase

      • Key-value stores

    • Redis
    • DynamoDB

      • Wide-column stores

    • Cassandra
    • HBase

      • Graph database

    • Neo4j
    • Amazon Neptune

  • NoSQL databases are designed for horizontal scaling, which involves adding more servers or nodes to the database system to handle larger volumes of traffic and data. This scalability makes NoSQL databases particularly suited to rapid growth and the varying demands ofmodern web applications.
In choosing between SQL and NoSQL, considerations include the specific requirements of the application, the complexity of the data relationships, scalability needs, and the expertise of the development team.

SQL databases are often preferred when transactional integrity (ACID properties: Atomicity, Consistency, Isolation, Durability) is paramount and the data structure is clear and unlikely to change frequently.

NoSQL databases, conversely, offer more flexibility and scalability, which can be beneficial in rapidly evolving environments where the data schema can change over time.

Understanding the differences between SQL and NoSQL databases is essential for anyone looking to efficiently store, manage, and analyze data. By aligning database selection with the specific requirements and challenges of each project, professionals can ensure robust, scalable, and effective data management solutions.

    Types of SQL Commands

  • Data Query Language (DQL)

    • SELECT


      perhaps the most frquently used SQL command. Allows you to query the db for specific info. you can use it to retrieve data from one or more tables, filter records with specific conditions, +more. It's the starting point for any data analysis task, enabling users to fetch and aggregate data as required

  • Data Definition Language (DDL)

    • CREATE


      creates new dbs, tables, or other db objects. Lays the foundation upon which data will be stores and managed

    • ALTER


      lets you modify the structure of an existing db object (ie. adding or deleting columns in a table or changing a column's data type)

    • DROP


      used to delete entire dbs, tables, or views. A powerful command that should be used with caution as it removes objects and their data permanently.

    • TRUNCATE


      used to delete all records from a table while leaving the table structure itself.

  • Data Manipulation Language (DML)

    • INSERT


      used to add new records to a table. It allows specifying which values will go into each column of the table for a new row.

    • UPDATE


      used to modify existing records ina table. Can change the values in one or more columns for all rows that meet certain criteria

    • DELETE


      similar to TRUNCATE, DELETE removes rows from a table. However, it offers more control, as you can specify exactly which rows to remove based on certain conditions.

  • Data Control Language (DCL)

    • GRANT


      allows specific users to perform specific tasks, like accessing, modifying, and executing particular transactions on the db

    • REVOKE


      takes back the permissions granted to the users, thus preventing them from performing certain operations on the db objects

  • Transaction Control Language (TCL)

    • COMMIT


      used to save all transactions made by DML statements like INSERT, UPDATE, or DELETE since the last COMMIT or ROLLBACK

    • ROLLBACK


      used to undo transactions that have not yet been committed, helping in error handling and ensuring data integrity

    • SAVEPOINT


      allows setting a point within a transaction to which you can later rollback, providing a way to partially undo changes within a larger transaction
